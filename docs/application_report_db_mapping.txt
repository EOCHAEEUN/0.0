FactoFit 신청서 PDF 리포트 DB 연결 명세
작성일: 2026-06-19

============================================================
1. 전체 연결 흐름
============================================================

frontend
  -> POST /api/reports/application.pdf
  -> 사용자 인증
  -> Supabase 테이블 조회
  -> 계산 및 보고서 문장 생성
  -> PDF 파일 반환

관련 코드
- frontend/src/pages/ApplicationDraftPage.tsx
- backend/app/routers/reports.py
- backend/app/services/application_report.py
- backend/app/core/database.py

API 요청 예시

{
  "company_id": "회사 UUID",
  "equipment_id": "설비 UUID",
  "policy_id": "정책 ID",
  "tone": "submission"
}

tone 값
- submission: 높임말 종결체
- analyst: 평서문 종결체
- nominal: 명사형 종결체

현재 프론트엔드는 tone을 생략하며 기본값 submission을 사용한다.

2. 직접 조회하는 Supabase 테이블
============================================================

1) company
- 조회 조건: company_id, 로그인 사용자의 user_id
- 용도: 신청기업 기본정보와 매출정보

2) equipment
- 조회 조건: company_id, equipment_id
- 용도: 설비 현황, 생산량, 운영비용

3) roi_output
- 조회 조건: company_id, equipment_id
- 정렬 조건: created_at 내림차순
- 사용 행: 최신 1건
- 용도: ROI 시나리오, 절감액, 업종 평균

4) matched_policy
- 조회 조건: company_id, equipment_id, 선택된 policy_id
- 정렬 조건: match_score 내림차순
- 사용 행: 최고 점수 1건
- 용도: 정책 적합도, 적격 여부, 추천 근거

5) policy
- 조회 조건: policy_id
- 용도: 정책 원문, 지원조건, 지원한도, 출처

6) draft_result
- 조회 조건: company_id, equipment_id, policy_id
- 정렬 조건: created_at 내림차순
- 사용 행: 최신 1건
- 현재 상태: 조회는 하지만 최종 PDF 본문에는 사실상 사용하지 않음

8. 자동 계산값
============================================================

1) 자기부담금

자기부담금 = 총 투자금 - 정부 지원금

2) 예상 회수기간

회수기간(개월) = scenario.payback_years * 12

3) 연간 운영비

연간 운영비 =
  equipment.energy_cost_annual
  + equipment.maintenance_cost_annual

4) 지원금 비율

지원금 비율 =
  scenario.subsidy_manwon
  / scenario.investment_manwon
  * 100

5) 적합도 점수 보정

matched_policy.match_score가 0보다 크고 1 이하이면 100을 곱한다.

예:
0.7 -> 70점

6) 지원금 대체값

scenario.subsidy_manwon이 없고 policy.max_amount가 있으면 다음 값을 사용한다.

min(
  scenario.investment_manwon,
  policy.max_amount
)

10. 인증 및 보안 연결
============================================================

백엔드는 다음 환경변수로 Supabase에 연결한다.

- SUPABASE_URL
- SUPABASE_SERVICE_KEY

연결 코드
- backend/app/core/config.py
- backend/app/core/database.py

리포트 생성 시 기업 조회 조건

company.company_id = API 요청 company_id
company.user_id = 로그인 사용자의 user_id

따라서 로그인 사용자가 소유한 회사만 조회할 수 있다.

주의
- .env 파일과 API 키는 Git 저장소에 커밋하지 않는다.
- BIZINFO_API_KEY와 DATA_GO_KR_API_KEY는 정책 수집 스크립트에서 사용한다.
- PDF 생성은 외부 정책 API를 직접 호출하지 않는다.
- PDF 생성 시점에는 Supabase에 저장된 policy 데이터를 조회한다.

13. 현재 확인사항
============================================================

- 실제 설비명 컬럼은 equipment.name이다.
- draft_result는 조회하지만 현재 최종 PDF 문장에는 사용하지 않는다.
- 정책 원문과 지원내용은 policy 테이블에 저장된 값을 사용한다.
- ROI 분석값은 roi_output.roi_data JSON에서 추출한다.
- 업종 평균값도 roi_output.roi_data.benchmark에서 추출한다.
- 높임말 보고서는 submission 문체를 사용한다.
- 높임말 본문은 합니다/습니다/입니다 계열의 단정형 종결 조건을 검증한다.
